# Web 测试界面 - 使用说明

## 📝 概述

这是一个用于测试 HuanVae Chat API 的网页测试工具，支持用户认证、好友管理和好友消息功能。

## 🚀 使用方法

### 1. 启动后端服务

```bash
cd /home/huanwei/Huanvae-Chat-Rust
cargo run
```

### 2. 打开测试页面

在浏览器中打开:
```
file:///home/huanwei/Huanvae-Chat-Rust/test-web/index.html
```

或者使用简单的 HTTP 服务器（推荐）:
```bash
cd /home/huanwei/Huanvae-Chat-Rust/test-web
python3 -m http.server 3000
# 然后访问 http://localhost:3000
```

## 📋 功能模块

### 1. 后端设置
- 配置 Auth API 地址（默认: `http://localhost:8080/api/auth`）
- 配置 Friends API 地址（默认: `http://localhost:8080/api/friends`）
- 配置 Messages API 地址（默认: `http://localhost:8080/api/messages`）

### 2. 用户认证
- **注册**: 创建新用户账号
- **登录**: 获取 Access Token 和 Refresh Token
- **查看个人信息**: 解析 JWT Token 查看用户信息
- **刷新 Token**: 使用 Refresh Token 获取新的 Access Token

### 3. 设备管理
- **查看设备列表**: 查看当前用户所有登录设备
- **删除设备**: 远程登出指定设备

### 4. 好友功能
- **发送好友请求**: 向其他用户发送好友申请
- **查看待处理请求**: 查看收到的好友申请
- **同意/拒绝申请**: 处理好友请求
- **查看已发送请求**: 查看自己发出的好友申请
- **查看好友列表**: 查看所有好友
- **删除好友**: 移除好友关系

### 5. 好友消息功能 ✨ (新增)

#### 发送消息
- **接收者ID**: 输入好友的用户ID
- **消息类型**: 
  - `text` - 文本消息
  - `image` - 图片消息
  - `video` - 视频消息
  - `file` - 文件消息
- **消息内容**: 输入要发送的内容
- **文件URL** (可选): 媒体消息时填写文件地址
- **文件大小** (可选): 文件大小（字节）

#### 获取消息列表
- **好友ID**: 查看与该好友的聊天记录
- **起始消息UUID** (可选): 用于分页，从指定消息之前查询
- **查询数量**: 默认50条，最大500条
- **消息显示**: 
  - 绿色背景 = 发送的消息
  - 蓝色背景 = 接收的消息
  - 显示发送时间、消息UUID
  - 提供删除和撤回按钮

#### 删除消息（软删除）
- **消息UUID**: 输入要删除的消息ID
- **效果**: 只对当前用户生效，对方仍可看到

#### 撤回消息
- **消息UUID**: 输入要撤回的消息ID
- **限制**: 
  - ⏰ 只能撤回2分钟内的消息
  - 🔒 只有发送者可以撤回
  - 🗑️ 撤回后双方都看不到

## 🎯 典型测试流程

### 测试好友消息功能

```
步骤1: 准备两个测试账号
  1. 注册用户A (user_id: alice, password: Password123)
  2. 注册用户B (user_id: bob, password: Password123)
  
步骤2: 登录用户A
  1. 输入 alice / Password123
  2. 点击"登录"
  3. Token 会自动保存到浏览器

步骤3: 用户A添加用户B为好友
  1. 在"好友功能"区域
  2. 目标用户ID输入: bob
  3. 点击"发起好友申请"
  
步骤4: 切换到用户B（可以开另一个浏览器窗口）
  1. 登录 bob / Password123
  2. 点击"获取待通过申请"
  3. 点击"同意"按钮
  
步骤5: 用户A发送消息给用户B
  1. 在"好友消息功能"区域
  2. 接收者ID输入: bob
  3. 消息内容输入: Hi Bob!
  4. 点击"发送消息"
  
步骤6: 用户B查看消息
  1. 好友ID输入: alice
  2. 点击"获取消息列表"
  3. 可以看到来自alice的消息
  
步骤7: 用户B回复
  1. 接收者ID输入: alice
  2. 消息内容输入: Hi Alice!
  3. 点击"发送消息"
  
步骤8: 测试删除和撤回
  1. 在消息列表中点击"删除"按钮
  2. 或在2分钟内点击"撤回"按钮
  3. 刷新消息列表查看效果
```

## 💡 提示

### 消息功能注意事项
1. **好友关系**: 只能向好友发送消息，非好友会返回错误
2. **Token 有效期**: Access Token 15分钟过期，需要刷新
3. **消息查询**: 使用分页查询可避免一次加载过多消息
4. **撤回时限**: 消息发送超过2分钟后无法撤回
5. **自动刷新**: 发送、删除、撤回成功后会自动刷新消息列表

### 调试技巧
1. **查看请求**: 每个操作都会显示完整的请求和响应
2. **本地状态**: 底部显示当前Token状态
3. **消息UUID**: 每条消息都显示UUID，可用于删除/撤回
4. **加载更多**: 如果有更多消息，会显示"加载更多"按钮

### 常见问题

**Q: 为什么无法发送消息？**
- 确保已登录（有Access Token）
- 确认双方已经是好友关系
- 检查接收者ID是否正确

**Q: 为什么看不到消息？**
- 确认好友ID输入正确
- 检查是否有消息被软删除
- 确认Token未过期

**Q: 撤回消息失败？**
- 只有发送者可以撤回
- 消息必须在2分钟内
- 确认消息UUID正确

## 📦 文件存储功能说明

### 文件上传
1. **采样SHA-256哈希计算**: 
   - **小文件（< 30MB）**：计算完整文件哈希
   - **大文件（≥ 30MB）**：使用**采样哈希**策略（文件元信息 + 开头10MB + 中间10MB + 结尾10MB）
   - 避免大文件内存溢出问题，3GB视频也能秒传
2. **分片上传**: 超大文件（>15GB）自动分片上传
3. **进度显示**: 实时显示上传进度
4. **秒传检测**: 文件已存在时跳过上传，直接返回UUID
5. **文件类型**: 支持用户图片、用户视频、用户文档

### 为什么使用采样哈希？
- **内存限制**: 浏览器无法一次性将3GB+文件加载到内存计算哈希
- **速度优化**: 采样哈希只需读取约30MB数据，几秒内完成
- **足够唯一**: 文件元信息（名称+大小+修改时间+类型）+ 三个采样点（开头/中间/结尾各10MB），已经能够唯一识别文件
- **秒传支持**: 相同文件的采样哈希相同，仍然可以实现秒传
- **服务端验证**: 采样哈希仅用于去重检查，不做完整性验证（大文件场景下可接受）

### 文件下载与预览（预签名URL机制）

#### 工作流程
1. **获取预签名URL**：
   - 前端通过UUID请求预签名URL
   - 后端验证权限后生成MinIO直连URL
   - 普通文件（< 15GB）：3小时有效期
   - 超大文件（≥ 15GB）：用户指定时间（3小时-7天）

2. **智能缓存**：
   - 前端缓存预签名URL，避免重复请求
   - 过期前5分钟自动提醒刷新
   - 缓存结构：`{ uuid: { url, expiresAt } }`

3. **文件类型判断**：
   - 根据文件名后缀判断类型
   - 支持：图片、视频、PDF、其他文件

4. **预览方式**：
   - **图片**：直接使用预签名URL作为img src
   - **视频**：使用预签名URL作为video src
     - ✅ 真正的流式播放
     - ✅ 支持拖动进度条
     - ✅ 支持Range请求
   - **PDF**：使用预签名URL作为iframe src
   - **其他**：提供下载链接

5. **超大文件特殊处理**：
   - 检测文件大小 > 15GB
   - 弹窗提示用户输入预计下载时间
   - 生成扩展有效期的预签名URL

#### 优势对比

| 特性 | 旧方案 (Bearer Token) | 新方案 (预签名URL) |
|------|----------------------|-------------------|
| **视频流式播放** | ❌ 需完整下载 | ✅ 真正流式，可拖动 |
| **后端压力** | ⚠️ 所有流量经过后端 | ✅ 直连MinIO |
| **浏览器缓存** | ❌ Blob URL不可缓存 | ✅ 可缓存 |
| **Range请求** | ❌ 不支持 | ✅ 支持 |
| **代码复杂度** | 较复杂（60-70行/函数） | 较简单（15-25行/函数） |
| **安全性** | ✅ 实时权限验证 | ⚠️ 时效性权限（过期前有效）|
1. **智能类型检测**: 
   - UUID访问的文件自动发送HEAD请求获取Content-Type
   - 根据文件类型自动选择预览或下载方式
   
2. **图片预览**:
   - 支持 jpg, jpeg, png, gif, webp 格式
   - UUID文件通过Bearer Token认证后加载
   - 普通URL直接预览
   
3. **视频播放**:
   - 支持 mp4, webm, ogg, mov 格式
   - UUID文件支持流式播放（可拖动进度条）
   - 大视频（>100MB）显示加载提示
   
4. **PDF查看**:
   - 内嵌iframe预览
   - UUID文件需要权限验证
   
5. **文件下载**:
   - UUID文件使用`downloadUuidFile()`函数，自动添加Authorization头
   - 下载时显示进度提示
   - 支持自定义文件名

### UUID访问机制
- 所有通过 `/api/storage/file/{uuid}` 访问的文件需要Bearer Token
- 前端自动从localStorage读取accessToken
- HEAD请求获取文件元信息（Content-Type、大小等）
- GET请求获取文件内容
- 支持Range请求（未来可支持视频真正的流式播放）
