# 用户认证系统文档

## 🔐 认证机制

### Token 双层设计

1. **Access Token（短期）**
   - 有效期：15分钟
   - 用途：每次API请求携带
   - 验证：使用RSA公钥验证
   - 存储：客户端内存/localStorage

2. **Refresh Token（长期）**
   - 有效期：7天
   - 用途：刷新Access Token
   - 存储：PostgreSQL数据库 `user-refresh-tokens` 表
   - 签名：使用RSA私钥

### 认证流程

```
用户登录
  ↓
生成 Access Token (15分钟) + Refresh Token (7天)
  ↓
客户端携带 Access Token 访问API
  ↓
Access Token 过期？
  ├─ 否 → 直接放行
  └─ 是 → 使用 Refresh Token 换取新的 Access Token
         ↓
      Refresh Token 过期？
         ├─ 否 → 返回新的 Access Token
         └─ 是 → 跳转登录页面
```

### 登出机制

1. 销毁当前设备的 Refresh Token
2. 将当前 Access Token 加入黑名单
3. 启用 `need-blacklist-check` 标志（15分钟）
4. 15分钟内所有请求都会检查黑名单

### 多设备支持

- Token 包含设备信息、MAC地址、设备标识
- 每个设备独立的 Refresh Token
- 支持远程撤销其他设备
- 可查看所有登录设备列表

### 安全特性

- ✅ RSA 2048位密钥对签名
- ✅ 智能黑名单检查（性能优化）
- ✅ 多设备管理
- ✅ Token 防篡改
- ✅ SQL注入防护
- ✅ 密码强度验证
- ✅ Bcrypt 密码加密

## 📡 API 端点列表

### 公开端点（无需认证）

| 方法 | 路径 | 功能 | 文件 |
|------|------|------|------|
| POST | `/api/auth/register` | 用户注册 | `handlers/register.rs` |
| POST | `/api/auth/login` | 用户登录 | `handlers/login.rs` |
| POST | `/api/auth/refresh` | 刷新Token | `handlers/refresh_token.rs` |

### 需要认证的端点（需要 Access Token）

| 方法 | 路径 | 功能 | 文件 |
|------|------|------|------|
| POST | `/api/auth/logout` | 用户登出 | `handlers/logout.rs` |
| GET | `/api/auth/devices` | 查看设备列表 | `handlers/revoke_device.rs` |
| DELETE | `/api/auth/devices/{device_id}` | 撤销指定设备 | `handlers/revoke_device.rs` |

## 📂 项目结构

```
src/auth/
├── 用户认证.md              # 认证方案文档 ✅
├── mod.rs                  # 模块导出
├── handlers/               # HTTP 请求处理
│   ├── mod.rs
│   ├── routes.rs           # 路由配置
│   ├── register.rs         # ✅ POST /api/auth/register
│   ├── login.rs            # ✅ POST /api/auth/login
│   ├── refresh_token.rs    # ✅ POST /api/auth/refresh
│   ├── logout.rs           # ✅ POST /api/auth/logout
│   └── revoke_device.rs    # ✅ GET/DELETE /api/auth/devices
├── middleware/             # 中间件
│   ├── mod.rs
│   └── auth_guard.rs       # ⭐ 鉴权中间件（验证Access Token + 黑名单检查）
├── services/               # 业务逻辑
│   ├── mod.rs
│   ├── token_service.rs    # ⭐ Token生成/验证/刷新
│   ├── blacklist_service.rs# ⭐ 黑名单管理
│   └── device_service.rs   # ⭐ 设备管理（查询/删除设备）
├── models/                 # 数据模型
│   ├── mod.rs
│   ├── user.rs             # 用户模型
│   ├── claims.rs           # ⭐ JWT Claims结构
│   ├── refresh_token.rs    # ⭐ Refresh Token模型
│   ├── device.rs           # ⭐ 设备信息模型
│   └── blacklist.rs        # ⭐ 黑名单Token模型
├── utils/                  # 工具函数

│   ├── crypto.rs           # ⭐ 密钥管理（加载/生成RSA密钥对）
```
