# 数据库表结构设计文档

## 📊 表名: users (用户数据表)

### 基础字段

| 字段名 | 类型 | 说明 | 存储内容结构 |
|--------|------|------|-------------|
| user-id | TEXT PRIMARY KEY | 登录id | 唯一标识 |
| user-nickname | TEXT NOT NULL | 昵称 | 用户显示名称 |
| user-password | TEXT NOT NULL | 密码 | 加密后的密码哈希 |
| user-email | TEXT UNIQUE | 邮箱 | 用户邮箱地址 |
| admin | TEXT | 是否是管理员 | 'true' 或 'false' |

### 好友关系字段

| 字段名 | 类型 | 说明 | 存储内容结构 |
|--------|------|------|-------------|
| user-owned-friends | TEXT | 拥有的好友列表 | friend-nickname: 好友昵称<br>friend-remark: 好友备注<br>friend-id: 好友id<br>add-time: 添加时间 |
| user-pending-friend-requests | TEXT | 等待通过的好友申请 | request-user-id: 申请用户id<br>request-message: 申请消息<br>request-time: 申请时间 |
| user-sent-friend-requests | TEXT | 发出的好友申请 | sent-to-user-id: 发送给用户id<br>sent-message: 申请消息<br>sent-time: 发送时间 |

### 群聊和对话字段

| 字段名 | 类型 | 说明 | 存储内容结构 |
|--------|------|------|-------------|
| user-joined-group-chats | TEXT | 进入的群聊 | join-time: 入群时间<br>group-name: 群名称<br>group-id: 群id<br>group-remark: 群备注 |
| user-ai-conversation-data | TEXT | 与AI的对话数据 | conversation-id: 对话id<br>conversation-title: 对话标题<br>create-time: 创建时间<br>last-message-time: 最后消息时间 |
| user-chat-data-with-friends | TEXT | 与好友的私聊数据 | message-sender-id: 消息发送人id<br>message-content: 消息内容<br>send-date: 发送日期<br>message-type: 消息类型 |

### 文件字段

| 字段名 | 类型 | 说明 | 存储内容结构 |
|--------|------|------|-------------|
| user-file-data-and-url | TEXT | 个人文件 | file-name: 文件名称<br>file-type: 文件类型<br>file-size: 文件大小<br>upload-time: 上传时间<br>file-path-url: 文件路径/URL |

### 黑名单检查控制字段

| 字段名 | 类型 | 说明 | 存储内容结构 |
|--------|------|------|-------------|
| need-blacklist-check | BOOLEAN DEFAULT false | 是否需要黑名单检查 | true: 需要检查<br>false: 跳过检查 |
| blacklist-check-expires-at | TIMESTAMP | 黑名单检查过期时间 | 15分钟后自动过期<br>过期后自动关闭检查 |

#### 使用说明

此字段用于智能性能优化，在以下安全事件时启用：
- 用户修改密码
- 远程登出其他设备
- 检测到 Token 泄露

**工作原理：**
1. 正常情况下 `need-blacklist-check = false`，跳过黑名单查询（高性能）
2. 安全事件发生时设置 `need-blacklist-check = true`，启用黑名单检查（15分钟）
3. 15分钟后自动恢复为 `false`（定时任务清理）

## 📊 表名: user-refresh-tokens (用户刷新Token管理表)

### 用途
管理多设备登录，每个设备独立的 Refresh Token（7天有效期）

### 字段设计

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| token-id | TEXT PRIMARY KEY | Token唯一ID | UUID v7 |
| user-id | TEXT NOT NULL | 用户ID | 外键关联 users.user-id |
| refresh-token | TEXT NOT NULL UNIQUE | 刷新Token | JWT格式，7天有效 |
| device-id | TEXT NOT NULL | 设备唯一标识 | 基于设备信息生成的哈希 |
| device-info | TEXT | 设备详细信息 | JSON格式存储 |
| ip-address | TEXT | 登录IP地址 | 用于安全监控 |
| created-at | TIMESTAMP | 创建时间 | Token签发时间 |
| expires-at | TIMESTAMP NOT NULL | 过期时间 | 7天后 |
| last-used-at | TIMESTAMP | 最后使用时间 | 每次刷新时更新 |
| is-revoked | BOOLEAN DEFAULT false | 是否已撤销 | 登出时设为true |
| revoked-at | TIMESTAMP | 撤销时间 | 撤销时记录 |
| revoked-reason | TEXT | 撤销原因 | user_logout, security_issue等 |

### device-info 字段存储格式（JSON）

```json
{
  "device_type": "mobile",           // "mobile", "desktop", "tablet"
  "os": "Android 14",                // 操作系统
  "browser": "Chrome 120",           // 浏览器
  "mac_address": "00:1A:2B:3C:4D:5E",// MAC地址（可选）
  "device_name": "Pixel 8 Pro",      // 设备名称
  "user_agent": "Mozilla/5.0..."     // 完整User-Agent
}
```

### 索引

- `idx-refresh-tokens-user-id` - 用于查询用户的所有设备
- `idx-refresh-tokens-token` - 用于快速验证Token
- `idx-refresh-tokens-expires` - 用于清理过期Token
- `idx-refresh-tokens-device` - 用于设备管理

## 📊 表名: token-blacklist (Token黑名单表)

### 用途
紧急撤销 Access Token（15分钟有效期内需要立即失效的情况）

### 字段设计

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| jti | TEXT PRIMARY KEY | Token的jti字段 | JWT中的唯一标识 |
| user-id | TEXT NOT NULL | 用户ID | 便于查询用户的所有黑名单Token |
| token-type | TEXT | Token类型 | "access" 或 "refresh" |
| expires-at | TIMESTAMP NOT NULL | Token过期时间 | 与原Token一致 |
| blacklisted-at | TIMESTAMP DEFAULT CURRENT_TIMESTAMP | 加入黑名单时间 | |
| reason | TEXT | 撤销原因 | 安全事件、密码修改等 |

### 索引

- `idx-blacklist-jti` - 主键索引
- `idx-blacklist-user-id` - 用于查询用户黑名单
- `idx-blacklist-expires` - 用于定期清理过期记录

### 清理策略

定期删除已过期的黑名单记录（因为过期的Token本身已经无效）：
DELETE FROM "token-blacklist" WHERE "expires-at" < NOW();

## 📝 数据存储格式说明

### TEXT 字段存储约定

所有复合数据以**逗号分隔的键值对**形式存储，格式：`key1:value1,key2:value2`

#### 示例：

```
# user-owned-firends 字段存储示例：
friend-nickname:张三,friend-remark:老同学,friend-id:user-123,add-time:2025-01-15T10:30:00

# group-members 字段存储示例：
member-id:user-001,member-nickname:李四,join-time:2025-01-10T08:00:00,member-role:member
```

### 多条记录存储

使用**分号(;)**分隔多条记录：

```
# 多个好友存储示例：
friend-nickname:张三,friend-remark:老同学,friend-id:user-123,add-time:2025-01-15;friend-nickname:李四,friend-remark:同事,friend-id:user-456,add-time:2025-01-16

# 多个群成员存储示例：
member-id:user-001,member-nickname:张三,join-time:2025-01-10,member-role:member;member-id:user-002,member-nickname:李四,join-time:2025-01-11,member-role:admin
```

### 查询示例

```sql
-- 查询用户的所有好友
SELECT "user-id", "user-owned-firends" 
FROM "users" 
WHERE "user-id" = 'user-001';

-- 查询群聊的所有成员
SELECT "group-id", "group-name", "group-members" 
FROM "groups" 
WHERE "group-id" = 'group-001';
```
